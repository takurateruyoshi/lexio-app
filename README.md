# レキシオ — オンライン P2P 対戦

麻雀×ポーカー×大富豪のカードゲーム「レキシオ」をブラウザで遊べるWebアプリ。
サーバー不要の完全静的サイトで、AI対戦もオンライン対戦もすべてブラウザ内で動きます。

## 遊び方

1. タイトルで「▶ 対戦する」→ 卓の人数（2〜5）と **AIの席数** を選ぶだけ。
   - AI以外の席が自分だけ → すぐソロ対戦
   - 募集席がある → 部屋が作られ、**招待リンク**（開くだけで参加）か部屋コードで友達が参加。
     開始時に空席はAIが埋めます。
   - **全席AI** → AI同士の対局を観戦・記録（連続対局・速度・精密思考・手札公開）
2. 名前を聞かれるのは複数人で遊ぶ時（ロビー/参加時）だけ。
3. 「📖 チュートリアル」でルール説明 → ヒント付き練習対局へ。
4. リロードしても復帰できます（ソロ=自動保存、オンライン=席トークンで自動再接続）。

オンライン対戦は WebRTC (PeerJS) による P2P 接続です。接続時のみ無料のシグナリング
サーバーを利用し、対局データはプレイヤー間で直接やり取りされます。
（対称NAT同士など、環境によっては接続できない場合があります。同じWi-Fi内なら確実です）

## Neoルール（スペシャルカード）

対戦設定で「Neo」を選ぶと、各プレイヤーにスペシャルカードが3枚配られます（3人以上・1ラウンド1枚まで）。
現在実装済み: **ジョーカー4色**（人数に応じて3〜5/3〜6/3〜7の代用。同じ色・数字の牌との同時出しのみ不可で、色違いならペアやフルハウスにも使えます）、
**新しい始まり**（配牌直し）、**失われた権利**（強制パス）、**不要なギフト**（牌を押し付け）、
**勝者独占**・**救いの手**（精算時・1位から順に使用機会）。
※ ダブルショット/致命的な運命/物々交換/フェアゲーム/下剋上はカード文言の確認後に追加予定。
※ AIはまだスペシャルカードを使用しません（学習対応は今後）。

## AI

全AIは同一の**正準モデル**（`model/weights.json` の θ）で手を選びます:

1. **ベイズ推定**: 出た牌と「パスした」という情報で条件付けし、相手の手札分布を
   棄却サンプリングで推定。
2. **モンテカルロ・ロールアウト**: 候補手ごとに「この手を出したらどうなるか」を
   サンプルした世界で終局までシミュレートし、期待収支が最大の手を選択。
   総ロールアウト予算は逐次淘汰（successive halving）で統計的に配分され、
   ゲーム状況に応じた人為的な条件分岐は持ちません。
3. 方策・枝刈りの定数はすべて θ に集約され、値は自己対戦学習が決めます。

ログには各AIの「勝率 / この役が通る確率 / 期待収支」と全候補のEV表が記録されます。

## 継続学習（常時・中央集約）

- `tools/train_evolve.mjs` が**二系統**で θ を学習します:
  自己対戦EVのみのCEM（系統A）と、収集牌譜の「勝った人間の着手との一致率」を
  ブレンドしたCEM（系統B、牌譜不足時は自動でEVのみにフォールバック）。
  最後に A・B・現行チャンピオンの3者アリーナで最も強い θ だけを
  `model/weights.json` に昇格させ、勝者を次世代の起点に繋ぎます
  （系統の最新候補は `model/weights_selfplay.json` / `model/weights_human.json`、
  経過は `model/history.json`）。
- GitHub Actions（`.github/workflows/train.yml`）が **6時間ごと**に実行し、
  昇格があれば `model/weights.json` を更新（Pages が自動再配信）。
  旧 `tools/train_selfplay.mjs`（自己対戦のみ）も残してあります。
- 全クライアントは同じ正準モデルを読み込むため、ブラウザごとにAIが分岐しません。

## 対局データの収集（研究用・二段構え）

- 対局終了時、匿名化した牌譜（名前は席番号に置換）が送信キューに入り、
  `model/collect.json` の設定に従って送信されます（フッタのトグルでオプトアウト可）。
  1. `httpUrl` が設定されていれば gzip バッチPOST（Cloudflare Worker 等。雛形と手順は
     `tools/cloudflare/README.md`）
  2. 未設定/失敗時は **P2P収集ピア**へ: [collector.html](collector.html) が
     開かれている間、牌譜がP2Pで届き蓄積されます（JSONLエクスポート可）。
- 送れなかった分はローカルに残り、定期的に再送されます。
- **自動収集**: GitHub Actions（`.github/workflows/collect.yml`）が毎時、
  headless Chrome で収集ピアを約50分間開き、受信した牌譜を
  `game-data` ブランチに JSONL として蓄積します（`tools/collector_headless.mjs`）。
  牌譜は匿名化済みですが、公開リポジトリのブランチに置かれる点に留意してください。
  学習ワークフローはこのブランチを参照します。

## AI観戦・記録モード（研究用）

- タイトルの「🔬 AI観戦・記録」から、全席AIの自動対局を観戦できます
  （人数・ラウンド数・連続対局数1〜100・速度・精密思考1秒/手・手札公開）。
- 全対局は牌譜（配牌・全手順・各手番の候補EV表・スコア）として IndexedDB に保存され、
  「牌譜をエクスポート」でJSONダウンロードできます（ソロ/オンラインの対局も保存）。
- **阻害行動の事後アノテーション**: 「残り枚数の少ないプレイヤーが追随できないサイズを
  リードした手」に `blocking` タグを付けます。これは記録の解析であり、AIの意思決定には
  一切影響しません — 連携的な阻害はEV最大化から創発したものだけが観測されます。

## ルール概要

- 牌は4スート（☁雲・★星・☾月・☀太陽）×数字。強さは 3 < 4 < … < 最大 < 1 < 2。
- 役: 単騎 / ペア / トリプル / 5枚役（ストレート < フラッシュ < フルハウス <
  フォーカード < ストレートフラッシュ）。同じ枚数でのみ上回れます。
  ストレートは上端ラップ（例: 7-8-9-1-2）も有効。
- パスしても、誰かが新しい役を出せばまた出せます。全員がパスすると場が流れ、
  最後に出した人がリード。
- 誰かが上がった瞬間にラウンド終了。残り枚数の差をペアワイズで支払い、
  手中の **2** 一枚ごとに支払いが ×2（1枚で2倍、2枚で4倍…）。

## 開発

静的サイトです。ローカルでは任意の静的サーバーで配信してください:

```bash
python3 -m http.server 8000
# → http://localhost:8000
```

- `js/engine.js` — ルールエンジン
- `js/ai.js` / `js/ai.worker.js` — ベイズ+ロールアウトAI（Web Worker実行）
- `js/game.js` — 権威的ゲーム進行とプレイヤー別ビュー
- `js/net.js` — PeerJS ルーム（ホスト権威型）
- `js/ui.js` / `js/main.js` — 描画と画面遷移

`app.py` / `game_cfr_agent.py` / `static/` は CFR・連携抑制の研究用Python実装
（旧版）で、Webアプリとは独立しています。
